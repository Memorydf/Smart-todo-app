# 智能待办小程序 — 完整项目日志

> 文档说明：本日志按需求与迭代顺序整理，涵盖前端（uni-app）、后端（Spring Boot）、AI（DeepSeek-V4-Flash API / Ollama）、数据与配置。日期以开发阶段为序，不绑定具体日历日。

---

## 一、项目目标

构建一套 **智能待办** 系统：**微信小程序客户端** + **Spring Boot 服务端** + **本地 Ollama 大模型**，实现待办的增删改查、统计展示、AI 对话，以及 **AI 根据自然语言生成多条待办并写入 MySQL**，前端列表自动刷新。

---

## 二、技术栈总览

| 层级 | 技术 |
|------|------|
| 小程序 / H5 | uni-app、Vue 3、Pinia、axios（自定义 uni.request 适配器）、SCSS |
| 后端 | Spring Boot 3、MyBatis-Plus、MySQL 8、Lombok、Jackson |

## 四、功能清单（当前实现）

### 4.1 小程序端

- **待办首页**：问候语、日期、统计卡片（今日待办 / 进行中 / 逾期 / 已完成）、本周日历条、分类筛选、隐藏已完成、待办列表（`TodoCard`）。
- **新建 / 编辑 / 详情**：标题、描述、分类、优先级、截止日期；详情支持勾选完成、编辑、删除。
- **统计页**：总量、完成率、环形进度、分类进行中分布（基于当前列表数据）。
- **AI 对话页**：调用后端 `POST /ai/chat`，失败时本地简短兜底回复。
- **设置页**：紧凑列表开关、清空待办、清空 AI 对话记录、关于与配置说明。
- **AI 一键生成待办**（`TODO_DATA_SOURCE=remote` 时）：首页「AI 生成」弹层 → `POST /ai/generateTodosAndSave` → 写库后 `GET /todo/list` 刷新。
- **紧凑列表**：设置开关控制 `TodoCard` 与首页列表区域更紧凑的样式（间距、字号等）。
- **数据来源**：`config.js` 中 `TODO_DATA_SOURCE` 为 `remote` 时走 Spring Boot + MySQL；`local` 时仅本地 `uni.storage`（无服务端写库、无「AI 生成」写库入口）。

### 4.2 后端

- **待办 CRUD**：`POST /todo/add`、`PUT /todo/update`、`DELETE /todo/delete/{id}`、`GET /todo/list`、`PUT /todo/finish/{id}`、`DELETE /todo/clear`。
- **统一响应**：`Result`（`code`、`msg`、`data`）。
- **全局异常**：`@RestControllerAdvice`，校验 / 参数 / 业务 / AI 调用等异常统一封装。
- **跨域**：`CorsConfig` 便于 H5 / 工具联调。
- **统计**：`GET /statistics/overview`（总量、完成/未完成、逾期、今日到期、分类未完成数）。
- **AI**：`POST /ai/chat`（落库 `ai_chat`）、`POST /ai/generateTodo`（纯文本建议）、`POST /ai/generateTodosAndSave`（解析 JSON 或行文本后批量插入 `todo`）。
- **AI**：`DeepSeekUtil` / `OllamaUtil` 调用；默认使用 DeepSeek-V4-Flash API，未配置 API Key 时回退到 Ollama |

### 4.3 数据库（概要）

- **todo**：标题、内容、优先级、状态、截止日期、分类、创建/更新时间等（详见 `backend/src/main/resources/db/schema.sql`）。
- **ai_chat**：问题、回答、创建时间。

## 五、迭代记录（按需求时间线）

### 阶段 A：前端方案落地（初版）

- 使用 **uni-app + Vue3 + Vite** 模板初始化工程。
- 实现页面：首页、新建/编辑/详情、AI 页、统计、设置；移除后的 **欢迎页** 曾存在，后按需求删除。
- **Pinia** 管理待办、AI 消息、设置；待办初版为 **纯本地存储**。
- 组件：`TodoCard`、`AiMessage`、`Loading`、`Empty`；Tab 栏与基础 UI（蓝色主色、卡片、圆角）。
- **api** 层预留 axios + `uni.request` 适配器。

### 阶段 B：欢迎页移除

- 启动页改为直接进入 **待办 Tab 首页**；删除欢迎页路由与相关存储键逻辑。

### 阶段 C：后端完整方案

- 新增 **backend** Maven 工程：Spring Boot 3、MyBatis-Plus、MySQL、Todo / AiChat 实体与 Mapper、Service、Controller。
- 配置 `application.yml`（数据源、MyBatis-Plus、Ollama 地址与模型名等）。
- 提供 `schema.sql` 建库建表。

### 阶段 D：AI 助手方案

- 默认使用 **DeepSeek-V4-Flash API**；支持回退到 Ollama；`OllamaClient` 整合为 **`OllamaUtil`**（符合方案中的工具类命名思路）。
- 优化对话 / 生成待办类 Prompt；剥离 `` / `` 等推理片段，避免直接展示给用户。
- 小程序 **`config.js`**：`API_BASE_URL`；**`request.js`** 统一处理 `code !== 200`。
- **`store/ai.js`**：`replyTo` 请求 `POST /ai/chat`，失败走本地兜底文案。

### 阶段 E：AI 智能待办助手（端到端写库 + 前端同步）

- 后端 **`POST /ai/generateTodosAndSave`**：AI 输出优先按 **JSON 数组** 解析为 `AiGeneratedTodoItem`，失败则 **按行降级**；事务内批量 `todoService.add`。
- 后端 **`DELETE /todo/clear`** 与 `TodoService.clearAll`。
- 前端 **`TODO_DATA_SOURCE`**：`remote` 时待办 **增删改查与列表** 对接 `/todo/*`，并在成功后镜像一份到本地 storage 作为断网时的降级展示；**首页「AI 生成」** 仅 `remote` 显示。
- 首页 / 统计 / 设置 **`onShow`** 在 `remote` 下补充 **`fetchList`**，保证从其他入口返回后列表与库一致。

### 阶段 F：紧凑列表生效

- **`TodoCard`** 根据 `settings.compactList` 增加 `compact` 样式类（内边距、间距、字号、勾选框尺寸等）。
- **首页** 列表容器在紧凑模式下略减上下留白。
- 设置文案更新为正式说明（去掉「预留」表述）。

## 六、配置说明（运维 / 开发）

| 配置项 | 位置 | 说明 |
|--------|------|------|
| 后端地址 | `src/utils/config.js` → `API_BASE_URL` | 小程序真机须填 **电脑局域网 IP:端口**，不可使用本机 `127.0.0.1` |
| 待办数据源 | `src/utils/config.js` → `TODO_DATA_SOURCE` | `remote`：MySQL + 接口；`local`：仅本地 |
| 数据库 | `backend/.../application.yml` | JDBC URL、用户名、密码 |
| AI 配置 | `application.yml` → `deepseek.api-key`、`deepseek.model` / `ollama.base-url`、`ollama.model` | 优先使用 DeepSeek API，未配置 API Key 时使用 Ollama |
| 微信 AppID | `src/manifest.json` → `mp-weixin.appid` | 发布 / 真机调试填写 |

## 七、构建与运行命令（摘要）

**小程序**

```bash
npm install
```

## 八、已知限制与后续可扩展方向

- **AI 输出稳定性**：若模型未严格输出 JSON，依赖行级降级，条目标题质量可能波动；可加强 Prompt 或改为服务端二次校验 / 重试。
- **统计页**：当前以前端 Store 聚合为主；可与 `GET /statistics/overview` 进一步对齐为纯服务端统计。
- **鉴权**：当前接口无用户体系，适合单用户 / 内网演示；多用户需加登录与 `user_id` 维度。
- **ECharts**：方案中曾提及图表扩展，当前统计为 CSS 环形图 + 条形图，可按需引入图表库。
